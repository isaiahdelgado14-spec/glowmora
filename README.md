# GlowMora Beauty & Makeup Artistry — Website

Luxury black + rose gold site for a traveling makeup artist in Austin, TX.
Pure HTML/CSS/JS — no build step, no dependencies. Double-click `index.html`
to open it in a browser.

## Pages & files

| File           | Purpose                                                     |
| -------------- | ----------------------------------------------------------- |
| `index.html`   | Home: hero, about (Fatimah), services, gallery, reviews     |
| `booking.html` | Availability calendar, time slots, booking request form     |
| `styles.css`   | All design — colors, fonts, layout (tokens in `:root`)      |
| `script.js`    | Shared: nav, scroll reveals, gallery filters                |
| `booking.js`   | Calendar, time slots, and form logic for the booking page   |

All decorative art — the rose-gold brush strokes, glitter splatter, and the
line-art face — is drawn in code (inline SVG sprite at the top of each HTML
file), so the site needs zero image assets to look finished.

## Important: what's demo/placeholder

- **Availability is fake.** "Booked" days and times in the calendar are
  generated by a placeholder formula in `booking.js` (see `isDayBooked` /
  `isSlotBooked`). Connect a real scheduler (Calendly, Square Appointments,
  Acuity) or edit those functions to reflect real availability.
- **The form doesn't send anywhere.** It shows a success message only. To
  receive real requests, create a free [Formspree](https://formspree.io) form,
  set `action="https://formspree.io/f/YOUR_ID" method="POST"` on the form tag
  in `booking.html`, and remove the submit handler at the bottom of
  `booking.js` (or convert it to a `fetch` POST).
- **Reviews are sample quotes** — replace with real client reviews.
- **Social links** point at the platforms' home pages except Instagram
  (`@GlowMora`); update hrefs in both HTML files.

## Customizing

- **Prices/services:** edit the cards in `index.html` (services section) and
  the Service Type dropdown in `booking.html`.
- **Colors:** all tokens live at the top of `styles.css` (`--rose-*`,
  `--black`, `--pink-band`, etc.). The SVG gradient colors are set in the
  `<linearGradient id="rg">` block at the top of each HTML file.
- **Photos:** the service cards, gallery tiles, about flatlay, and Instagram
  strip use code-drawn placeholders. Each has an HTML comment showing the
  one-line `<img>` swap (CSS already crops with `object-fit: cover`):
  - Service cards → replace `<div class="card-photo ...">` with
    `<img class="card-photo" src="images/bridal.jpg" alt="...">`
  - Gallery → replace `<div class="tile-art">` the same way
  - About → replace the `.flatlay` block with `<img class="about-photo" ...>`
- **Contact info:** phone/email/Instagram appear in the contact section,
  footer (both pages), policy card, announcement bar, and chat bubble
  (`sms:` link). Find-and-replace `(210) 559-1439` and `fatima006h@gmail.com`.

## Deploying (free options)

- **Netlify Drop** — drag this folder onto https://app.netlify.com/drop
- **GitHub Pages** — push to a repo, enable Pages in settings
- **Vercel** — `vercel` CLI or drag-and-drop in the dashboard

No build configuration is needed — it's a static site.
